## Overview

The `<color-input>` component supports three theme modes:

- **`auto`** (default) - Follows system preferences (prefers-color-scheme)
- **`light`** - Always uses light theme
- **`dark`** - Always uses dark theme

## Setting the Theme

Use the `theme` attribute:

```html
<color-input theme="auto"></color-input>
<color-input theme="light"></color-input>
<color-input theme="dark"></color-input>
```

Or set via JavaScript:

```js
picker.theme = 'dark';
```

## Inheriting from Your App

If you don't specify a theme, the component defaults to `auto`. This works well with app-level theme switching:

```html
<!-- App sets theme via data attribute -->
<html data-theme="dark">
  <!-- Picker automatically adapts -->
  <color-input value="oklch(70% 0.2 240)"></color-input>
</html>
```

## Dynamic Theme Switching

Change themes programmatically:

```js
const picker = document.querySelector('color-input');
const themeToggle = document.querySelector('#theme-toggle');

themeToggle.addEventListener('click', () => {
  picker.theme = picker.theme === 'light' ? 'dark' : 'light';
});
```

## Styling Theme-Specific Parts

Use CSS parts with theme-specific selectors:

```css
/* Light theme customization */
color-input[theme="light"]::part(panel) {
  box-shadow: 0 8px 32px rgba(0, 0, 0, 0.15);
}

/* Dark theme customization */
color-input[theme="dark"]::part(panel) {
  box-shadow: 0 8px 32px rgba(0, 0, 0, 0.5);
  border: 1px solid rgba(255, 255, 255, 0.1);
}

/* Auto adapts to system */
@media (prefers-color-scheme: dark) {
  color-input[theme="auto"]::part(panel) {
    box-shadow: 0 8px 32px rgba(0, 0, 0, 0.5);
  }
}
```

## Best Practices

:::tip
Use `theme="auto"` (the default) unless you have a specific reason to override. This provides the best user experience by respecting system preferences.
:::

:::note
The component's Shadow DOM encapsulation means internal styles are isolated. Theme changes only affect the component, not your page.
:::
