## Architecture

### Signals-Based Reactivity

The component uses [Preact Signals](https://preactjs.com/guide/v10/signals/) for fine-grained reactivity. Only affected DOM nodes re-render when state changes — no virtual DOM diffing, no framework overhead.

### Shadow DOM

All styles are encapsulated in Shadow DOM. Component styles never leak to the page, and page styles never break the picker. External customization is available through the `::part()` API.

### Area Picker

The 2D color area adapts its axes based on the active color space:

| Active color space | Internal model | X axis | Y axis |
| --- | --- | --- | --- |
| sRGB, Hex, HWB, Lab, OKLab | OkHSV | Saturation | Value |
| OKLCH, LCH | OKLCH | Chroma | Lightness |
| HSL | HSL | Saturation | Lightness |

Gradients are rendered at half resolution via an off-screen `<canvas>` and scaled up with CSS. Hue changes are batched with `requestAnimationFrame` to avoid redundant paints.

### Color Math

Color calculations are powered by [colorjs.io](https://colorjs.io) — accurate conversions between all supported spaces, automatic gamut mapping, and full compliance with CSS Color Module Level 4.

## Performance

The component targets minimal bundle impact: tree-shakeable ES modules, no framework dependencies beyond Preact Signals and colorjs.io, and ES2020+ output for smaller bundles.

At runtime, Signals prevent unnecessary re-renders, color conversions are cached, and the popover uses the native Popover API for hardware-accelerated positioning. Content is rendered on-demand when the popover opens.

## Popover Positioning

The popover automatically positions relative to the trigger button, respecting viewport boundaries and safe areas. It flips when near edges — no manual positioning required.

For custom placement, use `setAnchor()` to position relative to a different element:

```js
const picker = document.querySelector("color-input");
picker.setAnchor(document.querySelector("#my-anchor"));
picker.show();
```

## Gamut Detection

The read-only `gamut` property returns the smallest gamut that contains the current color:

| Gamut | Meaning |
| --- | --- |
| `srgb` | Works on all displays |
| `p3` | Requires Display P3 or better |
| `rec2020` | Requires ultra-wide gamut |
| `xyz` | Beyond Rec2020 |

A visual badge appears in the picker UI, and the `change` event includes `e.detail.gamut` for programmatic use.

## Form Integration

The component doesn't extend `HTMLInputElement`, but integrates easily with a hidden input:

```html
<form>
  <color-input id="picker" value="oklch(70% 20% 240)"></color-input>
  <input type="hidden" name="color" id="color-input" />
</form>

<script>
  const picker = document.querySelector("#picker");
  const hidden = document.querySelector("#color-input");

  hidden.value = picker.value;
  picker.addEventListener("change", (e) => {
    hidden.value = e.detail.value;
  });
</script>
```
